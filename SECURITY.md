# 安全策略

## 报告安全漏洞

如果你发现安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下方式私密报告：
- 发送邮件到仓库维护者

我们会在 48 小时内确认收到，并在 7 个工作日内给出初步评估。

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| master / develop | ✅ 最新功能，可能不稳定 |
| 最新 release | ✅ 推荐生产使用 |

## 安全最佳实践

- 始终验证外部输入的长度和格式
- 使用 `SyMemBackendAlloc` 而非标准 `malloc`
- 检查所有内存分配的 NULL 返回值
- 避免整数溢出（特别是 `count * sizeof(type)` 模式）
- 使用 `Systrcpy` 而非 `strcpy`/`sprintf`
